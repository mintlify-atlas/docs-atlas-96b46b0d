## The Two-Dimensional Scoring System

The Incel Test evaluates users on two independent axes:

1. **Looks Axis** (X-axis): Physical attributes and appearance
2. **Personality Axis** (Y-axis): Social behaviors and lifestyle choices

This creates a 2D coordinate system that places users into one of four quadrants, each representing a different "archetype."

<Note>
  The scoring system is implemented in `/assets/scripts/questioner.js:29-121`
</Note>

## The Scoring Algorithm

### Step 1: Answer Scoring

When a user selects an answer, the `calcScore()` function (line 109-121) converts it to a normalized value:

```javascript
function calcScore(answer, options) {
    n = options / 2;
    if ((options % 2) != 0) {
        // Handle odd number of answers by recursively treating as even
        if (answer < ((options + 1) / 2)) {
            return calcScore(answer, options + 1);
        } else {
            return calcScore(answer + 1, options + 1);
        }
    }
    // Even number of answers: scale to -1 to +1 range
    return ((answer - n) / n) * currentWeight;
}
```

**How it works:**

- For a question with 5 answers (indexed 0-4):
  - Answer 0: Most "chad" → negative score
  - Answer 2: Neutral → ~0 score  
  - Answer 4: Most "beta" → positive score
- The score is scaled by the question's weight
- Odd-numbered answer sets are normalized to even for consistent scoring

### Step 2: Accumulation

Scores accumulate separately for looks and personality (line 51-59):

```javascript
var score = calcScore(answerId, numAnswers);
if (isLooks) {
    looks += score;
} else {
    personality += score;
}
```

The system also tracks the total possible weight for each category:
- `looksCount`: Sum of all looks question weights
- `personalityCount`: Sum of all personality question weights

### Step 3: Normalization

After all questions are answered, the `finale()` function (line 123-191) normalizes scores to fit on a 253x253 pixel chart:

```javascript
// Scale scores to fit 253-pixel coordinate system
var personalityFinal = -1 * personality * (253 / personalityCount)
var looksFinal = looks * (253 / looksCount)
```

**Key points:**

- Personality is inverted (`-1 *`) to map correctly on the Y-axis
- Division by total weight normalizes to account for weighted questions
- Multiplication by 253 scales to pixel coordinates for SVG chart positioning

## Result Determination

The final coordinates determine which quadrant the user lands in (line 138-189):

```javascript
if (personalityFinal <= 0 && looksFinal >= 0) {
    // Beta Beta - worst quadrant
    result = "You are Beta Beta"
} else if (personalityFinal <= 0 && looksFinal <= 0) {
    // Beta Chad - good looks, poor personality  
    result = "You are Beta Chad"
} else if (personalityFinal >= 0 && looksFinal >= 0) {
    // Chad Beta - good personality, poor looks
    result = "You are Chad Beta"  
} else if (personalityFinal > 0 && looksFinal < 0) {
    // Chad Chad - best quadrant
    result = "You are Chad Chad"
}
```

<Warning>
  The coordinate system might seem counterintuitive: higher looks scores are actually "worse" (more beta), which is why looksFinal is not inverted but personalityFinal is.
</Warning>

## Weighted Questions Impact

The weighting system creates significant impact differences:

| Weight | Questions | Impact |
|--------|-----------|--------|
| 1 | 21 questions | Standard impact |
| 2 | 6 questions | 2x impact (height, build, dick length, anime, etc.) |
| 3 | 1 question | 3x impact (pornography preferences) |
| 5 | 1 question | 5x impact (sexual experience) |

**Example calculation:**

If all personality questions are answered neutrally except "Have you ever had sex?" (weight 5):

- Answering "No" adds +5 to personality score
- Total personality weight ≈ 25
- This single answer accounts for ~20% of personality score

```javascript
function moveDot(xOffset, yOffset) {
```

## Edge Cases

### Perfect Neutrality

If both scores equal exactly 0:

```javascript
if (personalityFinal == 0 && looksFinal == 0) {
    $("#congrats-message-header").prop('innerHTML', "You are Jared Fogle.");
}
```

This rare outcome triggers a special Easter egg result.

### Odd Answer Counts

Questions with odd numbers of answers (3 or 5 options) are handled recursively to maintain scoring consistency by treating them as if they had one additional phantom answer.

## Question Loading Flow

1. Document ready event fires (line 11)
2. `$.getJSON()` loads `questions.json` (line 19)
3. First question loads via `loadQuestion(0)` (line 24)
4. Progress bar maximum set to total questions (line 25)
5. User clicks answer → `logAnswer()` → `calcScore()` → `loadQuestion(next)`
6. After final question → `finale()` → result display
