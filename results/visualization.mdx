The Incel Test uses an SVG-based chart to visualize user results on a two-dimensional grid representing personality and looks scores.

## Chart Overview

The visualization is a **543×591px SVG chart** (`chad_chart.svg`) divided into four colored quadrants:

- **Top-left** (Orange): Beta Chad - Beta personality, Chad looks
- **Top-right** (Green): Chad Chad - Chad personality, Chad looks  
- **Bottom-left** (Red): Beta Beta - Beta personality, Beta looks
- **Bottom-right** (Blue): Chad Beta - Chad personality, Beta looks

## Coordinate System

The chart uses SVG coordinates with the origin at the **center of the chart**:

```
  Chad Looks (top)
       |
       |  Green    Orange
       |  (Chad    (Beta
       |   Chad)   Chad)
Beta --|---------- Chad
Personality    Personality
       |
       |  Blue     Red
       |  (Chad    (Beta
       |   Beta)   Beta)
       |
  Beta Looks (bottom)
```

### Key Coordinates

- **Chart dimensions**: 543px wide × 591px tall
- **Quadrant size**: 253.817px × 253.817px each
- **Center point**: (271.097, 293.291)
- **Vertical axis**: x = 271.097 (personality axis)
- **Horizontal axis**: y = 293.291 (looks axis)

### Coordinate Ranges

Scores are normalized and mapped to the chart:

<Tabs>
  <Tab title="Personality (X-axis)">

    - **Chad personality**: Positive values (right side)
    - **Note**: The raw personality score is inverted with `-1 *` before visualization
  </Tab>
  
  <Tab title="Looks (Y-axis)">
    - **Range**: -253 to +253  
    - **Chad looks**: Negative values (top side)

  </Tab>

## Score Normalization

Before visualization, raw scores are scaled to fit the chart dimensions:

```javascript
// Scale scores to chart dimensions
var personalityFinal = -1 * personality * (253 / personalityCount)
console.log("│  Final personality: " + personalityFinal);

var looksFinal = looks * (253 / looksCount)
console.log("│  Final looks: " + looksFinal);
```

**Location**: `questioner.js:130-133`

The scaling formula:
- Takes the raw accumulated score
- Divides by the total possible weight for that dimension
- Multiplies by 253 to fit the quadrant size
- Personality is inverted (`-1 *`) to match the coordinate system

## The `moveDot()` Function

The core visualization function moves a red dot indicator to the user's score position:

```javascript
function moveDot(xOffset, yOffset) {
    //var chart = document.getElementById("chart");
    //var doc = (chart.contentDocument) ? chart.contentDocument : chart.contentWindow.document;
    var dot = document.getElementById("pinpoint");

    if (dot) {
        var transformAttr = ' translate(' + xOffset + ',' + yOffset + ')';
        dot.setAttribute('transform', transformAttr);
    }
}
```

**Location**: `questioner.js:193-202`

### How It Works

<Steps>
  <Step title="Get the pinpoint element">
    Retrieves the SVG circle element with ID `pinpoint` from the DOM.
  </Step>
  
  <Step title="Build transform string">
    Creates an SVG `translate()` transform using the calculated offsets:
    ```javascript
    var transformAttr = ' translate(' + xOffset + ',' + yOffset + ')';
    ```
  </Step>
  
  <Step title="Apply transformation">
    Sets the `transform` attribute on the dot element, moving it relative to its original center position.
  </Step>
</Steps>

### Usage Example

The function is called in the `finale()` function after scores are calculated:

```javascript
moveDot((personalityFinal), (looksFinal));
$("#chart").removeAttr('hidden');
```

**Location**: `questioner.js:135-136`

<Note>
The chart is initially hidden with the `hidden` attribute and only revealed after the dot has been positioned to the final score.
</Note>

## Chart Labels and Axes

The SVG includes text labels for orientation:

- **Horizontal axis labels**: "Beta Personality" (left) and "Chad Personality" (right)
- **Vertical axis labels**: "Chad Looks" (top) and "Beta Looks" (bottom)
- **Quadrant labels**: Each quadrant displays its result type (e.g., "Beta Beta", "Chad Chad")

## Integration with Results

The visualization workflow:

<Steps>
  <Step title="User completes quiz">
    All questions answered, raw scores accumulated.
  </Step>
  
  <Step title="Scores normalized">
    The `finale()` function scales scores to chart dimensions.
  </Step>
  
  <Step title="Dot positioned">
    `moveDot()` translates the pinpoint to the calculated position.
  </Step>
  
  <Step title="Chart revealed">
    The hidden attribute is removed, displaying the chart.
  </Step>
  
  <Step title="Result card shown">
    The appropriate result description appears based on quadrant.
  </Step>
</Steps>

<Tip>
The chart is embedded directly in the HTML (not loaded as an external file) to ensure the `pinpoint` element is immediately accessible for manipulation.
</Tip>
